# ccusage

이 스킬은 Claude Code의 `stats-cache.json`을 분석하여, 현재 활성화된 모델과 토큰 사용량을 사용자에게 보고하고, 작업 전후의 토큰 변화량을 계산해 줍니다.

어시스턴트(당신)는 이 스킬을 이용해 `Bash` 명령어를 실행하고, 그 결과를 깔끔하게 정리해 사용자에게 알려주어야 합니다.

## 스크립트 실행 방법

컨테이너 내의 `Bash` 도구를 사용하여 노드 스크립트를 실행합니다. 상황에 맞춰 `start` 또는 `end` 인자를 붙일 수 있습니다.

### 1. 일반 조회 (작업 전/후 명시 없이 확인 시)
```bash
node /home/node/.claude/skills/ccusage/scripts/get-usage.cjs
```
- 결과를 바탕으로 현재 누적된(혹은 현재) 토큰 사용량을 한 눈에 볼 수 있게 사용자에게 보고합니다.

### 2. 작업 시작 시 사용량 저장 (스냅샷)
사용자가 작업을 시작하기 전 "ccusage로 체크해줘", "토큰 사용량 저장해줘"라고 요청하면 다음 명령을 실행하세요.
```bash
node /home/node/.claude/skills/ccusage/scripts/get-usage.cjs start
```
- 명령어 실행 후, 사용자에게 "스냅샷을 저장했습니다. 작업을 완료한 뒤에 끝났다고 알려주세요." 라고 안내합니다.

### 3. 작업 종료 후 사용량 분석 (결과 확인)
사용자가 작업이 완료되었다고 하거나, 아까 `start` 해둔 상태에서 "사용량 다시 알려줘", "ccusage 결과"라고 요청하면 다음 명령을 실행합니다.
```bash
node /home/node/.claude/skills/ccusage/scripts/get-usage.cjs end
```
- 명령어 실행 후, 아까 저장한 스냅샷과 현재의 차이(이번 작업에 사용한 토큰)를 친절하게 계산해서 알려줍니다. 만약 여러 모델이 출력되었다면, 주요 모델(이번 작업에서 쓰인 모델) 위주로 정리하세요.

## 규칙 및 권장사항
- 어시스턴트는 직접 통계를 계산할 필요 없이, 오직 노드 스크립트의 출력 결과를 기반으로 응답을 구성하면 됩니다.
- 보고서를 작성할 때는 마크다운 표나 목록을 사용하여 가독성을 높이세요.
