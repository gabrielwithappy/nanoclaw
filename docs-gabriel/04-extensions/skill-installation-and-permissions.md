# 스킬 설치 방법과 위치별 권한 특징 가이드

> 메신저(WhatsApp 등)를 통해 대화하는 Claude에게 새로운 기능을 추가하는 방법과, 각 설치 위치가 가지는 권한(Permission) 및 특징을 설명합니다.

> **참고:** 스킬 시스템의 아키텍처와 동작 원리에 대한 자세한 설명은 [시스템 명세서의 컨테이너 스킬 시스템](./skills-spec.md#container-skills-system) 섹션을 참조하세요.

---

## 1. 스킬 설치 위치 및 권한 연관관계

스킬을 어느 위치에 설치하느냐에 따라 해당 스킬을 사용할 수 있는 **주체(그룹)** 와 보안/마운트 **권한** 범위가 완전히 달라집니다. 스킬의 목적에 맞게 알맞은 설치 경로를 선택해야 합니다.

| 권한 종류 | 위치 | 사용 권한 범위 | 설명 및 특징 |
| :--- | :--- | :--- | :--- |
| **글로벌 컨테이너 스킬** | `container/skills/` | 모든 채팅방 에이전트 | 시스템에 등록된 모든 대화방(그룹)의 에이전트들이 공통으로 복제받아 사용할 수 있는 범용 도구입니다. 누구나 사용할 수 있으므로 민감한 접근 권한이 없는 범용적인 목적(예: 계산기, 브라우저)에 적합합니다. |
| **특정 그룹(방) 전용 스킬** | `data/sessions/{그룹명}/.claude/skills/` | 해당 `{그룹명}` 에이전트만 | 메인(`main`) 방 혹은 특정 단톡방에서만 단독으로 사용할 비밀/특권 스킬을 직접 꽂아 넣습니다. 다른 방의 에이전트는 절대 이 폴더에 접근할 수 없습니다. 중요한 정보나 관리 권한이 필요한 스킬은 이곳에 두어야 안전합니다. |
| **호스트 관리용 스킬** | `.claude/skills/` | 관리자 (사람 또는 시스템) | 챗봇 사용자가 아닌, 시스템을 세팅하는 "개발자(호스트)"가 쓰는 관리 명령어 묶음입니다. |

### 위치별 특징과 권한 마운트의 작동 원리
- **자동 복사 규칙**: `container/skills/`에 스킬을 추가하면, 새로운 컨테이너가 뜰 때마다 각 그룹의 고유 세션 폴더(`data/sessions/{그룹명}/.claude/skills/`)로 자동 동기화(복사)됩니다. 
- **격리된 권한 통제**: 에이전트가 실행되는 샌드박스 컨테이너는 자기 그룹만의 스킬 폴더만을 `~/.claude/` 내부로 마운트하여 받아옵니다. 이를 통해 서로 다른 채팅방이 타 그룹의 스킬과 권한을 침범할 수 없도록 강제 분리합니다.

## 2. 스킬 설치 방법 가이드 (4단계)

### 1단계: 목적별 위치를 선택해 스킬 폴더 생성

스킬의 접근 권한 목표에 맞추어 디렉터리를 생성합니다.

**방법 A. 안전하고 범용적인 공통 스킬 (Global 권한)**
```bash
mkdir container/skills/my-tool
```

**방법 B. 보안이 필요한 특정 그룹 전용 (Private 권한)**
```bash
mkdir -p data/sessions/main/.claude/skills/my-tool
```

### 2단계: SKILL.md 파일 생성 및 작성

해당 폴더 내부에 `SKILL.md` 이름으로 명세서를 작성합니다. **Claude는 오직 이 문서의 `description`과 사용 예제(Usage)를 읽고, 권한을 위임받은 에이전트로서 언제 도구를 써야 할지를 학습**합니다.

```markdown
---
name: my-tool
description: >
  사용자가 "날씨 알려줘", "기온 등 확인"을 요청할 때 사용하는 전용 스킬입니다.
  그룹 권한 내에서만 외부 API를 호출하여 날씨 정보를 가져올 수 있게 허락합니다.
---

# My Tool 가이드

## 사용법

해당 마운트된 컨테이너 환경에서 Bash 도구로 날씨 API를 호출할 권한이 주어집니다.

```bash
curl "https://api.weather.example.com/current?city=Seoul"
```

응답 JSON에서 `temperature` 필드를 추출해 사용자에게 전달해주세요.
```

### 3단계: 기반 프로그램 설치하기 (선택 사항)

만약 새로 만든 스킬이 jq, python3 등의 특정 리눅스 프로그램 패키지를 요구한다면 호스트가 아닌 컨테이너 이미지 `Dockerfile`에 설치 권한을 주어야 합니다.

```dockerfile
# container/Dockerfile 수정
RUN apt-get update && apt-get install -y jq python3-pip
```

이후 변경된 프로그램으로 권한 이미지를 새로 빌드합니다:
```bash
./container/build.sh
# 또는
docker build -t nanoclaw-agent:latest container/
```

### 4단계: 변경된 권한과 스킬 시스템 반영

추가된 스킬은 다음번 챗봇이 재시작하여 새로운 세션 컨테이너가 뜰 때 자동으로 반영됩니다. 이미 켜진 봇에 즉각 권한을 부여하고 싶다면 서비스를 강제로 재실행합니다.

```bash
# macOS 서비스 데몬 리로드
launchctl kickstart -k gui/$(id -u)/com.nanoclaw

# Linux 계열 리로드
systemctl --user restart nanoclaw
```

## 3. SKILL.md 잘 쓰는 요령

- **`description`은 트리거입니다**: Claude는 이 필드를 읽고 스킬 호출 여부를 결정합니다. 어떤 키워드가 나오면 이 권한을 쓸 것인지 최대한 구체적으로 명시하세요.
- **예제는 항상 실행 가능한 코드로**: Claude에게 동작 원리를 길게 설명하는 것보다, 실제 권한상 허용된 Bash 커맨드 한두 줄이나 파이썬 코드 예제를 제공하는 것이 100배 효과적입니다.

## 4. 트러블슈팅

**스킬이 인식되지 않을 때:**
- 파일명이 반드기 `SKILL.md`처럼 모두 대문자인지 확인하십시오.
- 해당 그룹이 이 스킬 폴더를 직접 읽을 마운트 권한이 생성되었는지 확인하십시오.
- `.../skills/{스킬이름}/SKILL.md`의 디렉터리 구조 규칙이 완벽한지 검시하십시오.

**도구 실행 실패 (명령을 찾을 수 없음):**
- 스킬에서 쓰는 도구가 실제 컨테이너 내부에 설치 권한 부여 및 빌드가 되었는지 살펴보십시오:
```bash
# 직접 샌드박스로 침투하여 권한 체크하기
docker run --rm -it --entrypoint /bin/bash nanoclaw-agent:latest
which jq
```
